## BỔ SUNG: LƯU PAYOS CREDENTIALS TRONG ADMIN SETTINGS (không dùng .env)

Thay vì đọc PAYOS_CLIENT_ID, PAYOS_API_KEY, PAYOS_CHECKSUM_KEY từ biến môi trường,
hãy lưu 3 giá trị này vào bảng `settings` và cho admin cấu hình qua giao diện.

---

### 1. SEED SETTINGS MẶC ĐỊNH

Thêm vào bảng settings các key sau (value để trống ''):
- payos_client_id
- payos_api_key
- payos_checksum_key
- monthly_price (đã có, giữ nguyên default '299000')

---

### 2. BACKEND — ĐỌC PAYOS CREDENTIALS TỪ DB

Tạo helper function `getPayosClient()` trong api-server:
- Đọc payos_client_id, payos_api_key, payos_checksum_key từ bảng settings
- Nếu bất kỳ giá trị nào còn trống → throw Error('PayOS chưa được cấu hình')
- Khởi tạo và trả về PayOS instance từ @payos/node với 3 key đó
- Gọi hàm này mỗi lần cần dùng PayOS (không cache, để admin đổi key có hiệu lực ngay)

Dùng getPayosClient() trong route POST /api/studio/payment/create-order
và route POST /api/webhooks/payos thay vì đọc từ process.env.

---

### 3. ADMIN API

Trong GET /api/admin/settings — bao gồm cả 4 key trên trong response.

Trong PUT /api/admin/settings/:key — cho phép update payos_client_id, payos_api_key,
payos_checksum_key, monthly_price (chỉ admin JWT).

Lưu ý bảo mật: khi trả về payos_api_key và payos_checksum_key trong GET,
che bớt giá trị (chỉ hiện 6 ký tự cuối, ví dụ "••••••abc123") — 
nhưng vẫn lưu full value trong DB và dùng full value khi gọi PayOS.

---

### 4. ADMIN FRONTEND — TRANG CÀI ĐẶT

Trong trang Admin, thêm section "Cấu hình PayOS" với giao diện tương tự ảnh đính kèm:

Layout:
- Tiêu đề: "Thông tin cấu hình PayOS"
- 3 field theo thứ tự: Client ID → Api Key → Checksum Key
- Mỗi field có:
  - Label
  - Input type="password" (mặc định ẩn, có icon toggle hiện/ẩn)
  - Icon copy để copy giá trị
  - Placeholder hiển thị giá trị đã che (nếu đã có key, hiện dạng ••••••abc123)
- Nút "Lưu cấu hình PayOS" → gọi PUT cho từng key đã thay đổi
- Nếu chưa cấu hình: hiển thị badge đỏ "Chưa cấu hình"
- Nếu đã cấu hình đủ 3 key: hiển thị badge xanh "Đã cấu hình"

Cũng trong trang này, đặt luôn section "Giá thuê bao":
- Input số "Giá hàng tháng (VNĐ)"
- Hiển thị preview định dạng: "299.000 ₫ / tháng"
- Nút Lưu riêng

---

### 5. XỬ LÝ KHI PAYOS CHƯA CẤU HÌNH

Khi studio gọi POST /api/studio/payment/create-order mà PayOS chưa cấu hình:
- Trả về HTTP 503 với { code: 'PAYOS_NOT_CONFIGURED', message: 'Hệ thống thanh toán chưa sẵn sàng, vui lòng liên hệ quản trị viên' }

Frontend studio: nếu nhận được lỗi này → hiển thị thông báo thay vì QR.

---

### LƯU Ý:

  (giữ lại làm tài liệu tham khảo, nhưng code không đọc từ đó nữa)
- Toàn bộ UI tiếng Việt
