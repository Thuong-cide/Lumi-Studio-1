### Who uses this feature
- **User**: Studio staff/editor (not the client)
- **Device**: Studio desktop computer (Windows/Mac) running Chrome or Edge

---

### Workflow (studio perspective)
1. Client finishes selecting photos on their device → exports a file name list
2. Studio staff receives that file name list (already in app state from "Chọn Ảnh" tab)
3. Staff opens "Lọc Ảnh" tab on the STUDIO COMPUTER
4. Staff points to the source folder (memory card, D:/DCIM, external drive, etc.)
5. Staff chooses where to copy files (editing folder, project folder, or new subfolder)
6. Staff selects format (JPG proof copies OR RAW originals for editing)
7. App copies matching files automatically
8. Staff gets a report of what was copied and what's missing

---

### Key difference from client-facing tool
- Uses File System Access API to read LOCAL DISK directly (not file upload)
- Staff should NEVER have to manually upload files — they work with local paths
- The tool accesses the computer's file system like a desktop app
- Large batches expected: 200–1000+ files per session
- RAW formats are primary use case (NEF, CR2, CR3, ARW, etc.)
